# API Keys Reference

Detailed reference for managing API keys via `nvcf-cli`.

## Overview

API keys are used for user-level operations: function invocation, listing functions, and queue monitoring. They are distinct from the admin JWT token (generated by `nvcf-cli init`) which is used for management operations like create, deploy, and delete.

## Generating API Keys

```bash
# Generate with defaults (24h expiration, default scopes)
nvcf-cli api-key generate

# Custom expiration
nvcf-cli api-key generate --expires-in 48h

# Custom description
nvcf-cli api-key generate --description "Development testing key"

# Custom scopes
nvcf-cli api-key generate --scopes invoke_function,list_functions

# Generate and validate
nvcf-cli api-key generate --validate

# All options
nvcf-cli api-key generate \
  --expires-in 7d \
  --description "Custom API key" \
  --scopes invoke_function,list_functions \
  --validate
```

### Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--expires-in` | `24h` | Expiration duration (e.g., `24h`, `7d`, `168h`) |
| `--description` | `"Generated by nvcf-cli"` | Key description |
| `--scopes` | Default scopes (see below) | Comma-separated scopes |
| `--validate` | `false` | Validate the key after creation |

### Default Scopes

When no `--scopes` flag is provided, these scopes are applied automatically:

| Scope | Description |
|-------|-------------|
| `invoke_function` | Execute deployed functions |
| `list_functions` | View available functions |
| `list_functions_details` | View detailed function metadata |
| `queue_details` | Monitor function execution queues |
| `manage_registries` | Manage registry credentials |

### Additional Scopes

Extended scopes available if your account has the necessary permissions:

| Scope | Description |
|-------|-------------|
| `register_function` | Register new functions |

The generated API key is automatically saved to the CLI state file and used for subsequent operations.

## Listing API Keys

```bash
nvcf-cli api-key list
```

Displays all API keys with their IDs, descriptions, creation dates, expiration dates, and status.

## Showing Current API Key

```bash
nvcf-cli api-key show
```

Shows the current saved API key from the CLI state file.

## Deleting API Keys

```bash
# Delete by ID
nvcf-cli api-key delete <key-id>

# Force delete without confirmation
nvcf-cli api-key delete <key-id> --force
```

Once deleted, the API key is no longer valid for authentication and cannot be recovered.

## Revoking API Keys

```bash
nvcf-cli api-key revoke <key-id>
```

Functionally identical to `delete` -- revokes the key permanently.

## Clearing API Keys

### Clear current key from state

```bash
nvcf-cli api-key clear
```

Removes the saved API key from the CLI state file without deleting it from the server. The key remains valid but is no longer used by the CLI automatically.

### Clear all keys for an owner

```bash
nvcf-cli api-key clear-all
```

Bulk operation that deletes all API keys associated with the current owner from the server.

## State Storage

Generated API keys are stored in `~/.nvcf-cli-state.json`. The CLI automatically loads the saved key for operations that require `NVCF_API_KEY`.

You can also set the API key via environment variable or config file:

- **Environment variable**: `NVCF_API_KEY`
- **Config file key**: `api_key`
