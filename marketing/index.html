Agent Architectures

The Right Agent for Every Task

Daedalus implements all four NeMo Agent Toolkit agent patterns, selectable per workflow or surfaced through the UI's mode selector.

⚡ Tool-Calling Agent

Fast, Schema-Driven Execution

Leverages native function-calling for structured, low-latency interactions. Ideal for well-defined tasks where tool selection and routing can be handled efficiently by the model.

Lowest latency for single or few tool calls
Structured tool input/output via schema routing
Error handling with configurable retries
YAML-defined tool names and LLM binding
Best for: Document Q&A, image processing, structured lookups, API integrations
🧠 ReAct Agent

Adaptive Multi-Step Reasoning

Implements the Think → Act → Observe loop, enabling dynamic decision-making between tool invocations. Surface as "Deep Thinker" mode for complex, open-ended research tasks.

Iterative reasoning adapts based on tool results
Handles ambiguous, multi-constraint problems
Intermediate steps visible to end users
Custom prompt with preserved ReAct format
Best for: Research tasks, competitive analysis, open-ended investigation
📋 ReWOO Agent

Plan Once, Execute Efficiently

Separates planning from execution — generates a complete plan upfront, then executes without re-querying the planner. Significantly reduces token usage compared to ReAct for predictable multi-step workflows.

Single planning pass, parallel execution
Lower token cost than ReAct for known workflows
Customizable planner and solver prompts
Tool schema in descriptions for disambiguation
Best for: Structured research pipelines, report generation, batch processing
💡 Reasoning Agent

Upfront Planning Layer

Wraps any existing agent or function with a dedicated reasoning step. A reasoning-capable model generates a structured plan that guides the underlying agent's execution.

Composable — wraps Tool-Calling or ReAct agents
Separates planning reasoning from execution
Requires reasoning-capable model (think tags)
Configurable reasoning and instruction templates
Best for: Complex tasks needing structured upfront reasoning before tool use

NAT Builder Packages

9 Custom NeMo Agent Toolkit Functions

Each capability is packaged as a standalone, reusable NAT function — independently testable, independently deployable, and usable across any NeMo Agent Toolkit workflow.

01

image_generation

Text-to-image generation with configurable output dimensions (768–1344px) and base64 JPEG encoding for inline chat display.

02

image_comprehension

Vision-language analysis for OCR, object detection, image description, and visual Q&A on user-uploaded images.

03

image_augmentation

Text-guided image editing using Flux Kontext API with support for 8 dimension pairs and automatic source image resizing.

04

nat_nv_ingest

Batch PDF processing pipeline using NVIDIA NV-Ingest with Milvus vector storage and intelligent 7-day Redis caching.

05

smart_milvus

Semantic retrieval with dynamic collection selection, distance cutoff filtering, and NVIDIA reranker model integration.

06

rss_feed

Intelligent RSS parsing with 4-hour caching, AI-powered reranking by query relevance, and automatic web scraping for full article content.

07

webscrape

JavaScript-capable web content extraction with Cloudflare bot protection handling and automatic conversion to clean Markdown.

08

vtt_interpreter

Converts VTT/SRT transcripts into structured meeting notes with organized sections for attendees, business updates, technical updates, and action items.

09

nat_helpers

Shared geolocation and utility functions used across the builder packages for consistent behavior and code reuse.

Deployment

From Laptop to Production Cluster

A single Docker image to try it locally. A production Helm chart for everything else.

Docker Compose

Full local stack in one command — frontend, dual backends, NGINX, Redis, and Milvus pre-configured.

  • All services containerized and orchestrated
  • Pre-configured NGINX reverse proxy
  • Redis Stack with RedisJSON + RedisSearch
  • Environment variable configuration via .env
  • Hot-reload for frontend development
# Clone and configure
cp .env.example .env
# Launch everything
docker compose up

Kubernetes / Helm

Production-grade chart with dual backend support, persistent storage, health monitoring, and TLS ingress.

  • Dual backend: Tool-Calling + ReAct agents
  • Configurable node placement and resource limits
  • Persistent storage classes for Redis and Milvus
  • Built-in health check CronJob with Slack alerts
  • TLS ingress with cert-manager support
  • JupyterLab integration for data exploration
# Deploy to cluster
helm upgrade --install daedalus \
  ./helm/daedalus \
  -f values.yaml
